# 端到端测试清单

> 本清单覆盖能源审计平台全部三端业务流程及系统能力，用于交付验收前的完整功能验证。

---

## 一、企业端流程 (Enterprise)

### 1.1 企业注册与准入

- [ ] 企业提交准入申请（填写企业名称、统一社会信用代码、行业、联系人等）
- [ ] 申请提交后状态为 `pending_review`（待审核）
- [ ] 企业端可查看申请状态
- [ ] 被驳回后可修改重新提交

### 1.2 企业基础配置

- [ ] 能源品种定义（电力、煤炭、天然气、柴油、蒸汽等）
  - [ ] 新增能源品种（名称、类型、折标系数、计量单位）
  - [ ] 编辑已有能源品种
  - [ ] 停用/启用能源品种
- [ ] 产品定义
  - [ ] 新增产品（名称、计量单位、工序关联）
  - [ ] 关联用能单元
- [ ] 用能单元定义
  - [ ] 新增单元（名称、类型、能源消耗边界描述）
  - [ ] 关联能源品种
- [ ] 配置完整性校验（未完成配置时阻止进入填报阶段）

### 1.3 数据填报（24个模块）

- [ ] 模块列表展示（根据业务类型筛选可见模块）
- [ ] 进入单个填报模块
- [ ] 表单/表格渲染正确
- [ ] 数据保存（draft → saved）
  - [ ] 保存时允许存在校验警告
  - [ ] 保存成功提示
- [ ] 数据提交（saved → pending_submit → submitted）
  - [ ] 提交时执行5层校验
  - [ ] 存在阻断性错误时阻止提交
  - [ ] 提交成功后状态变更
- [ ] 数据退回处理（submitted → returned）
  - [ ] 查看退回原因
  - [ ] 修改后重新提交

### 1.4 校验规则执行

- [ ] 第1层：基础数据类型校验
- [ ] 第2层：字段级约束校验（必填、范围、格式）
- [ ] 第3层：跨字段校验（能源平衡等）
- [ ] 第4层：跨模块校验
- [ ] 第5层：完整性校验
- [ ] 校验结果展示（错误/警告/信息三级）
- [ ] 修正建议展示

### 1.5 计算引擎

- [ ] 字段级自动计算触发
- [ ] 模块级汇总计算
- [ ] 综合能耗（TCE）计算
- [ ] 碳排放计算（活动数据 x 排放因子 x 氧化率）
- [ ] 计算结果快照保存

### 1.6 报告查看与修订

- [ ] 查看系统生成的报告初稿
- [ ] 下载报告文件
- [ ] 上传企业修订稿
- [ ] 报告版本历史查看

### 1.7 整改任务处理

- [ ] 接收整改任务通知
- [ ] 查看整改任务详情（来源问题、截止日期）
- [ ] 认领整改任务（pending_claim → in_progress）
- [ ] 更新整改进度（百分比、说明、附件）
- [ ] 提交整改验收（in_progress → pending_acceptance）

## 二、管理端流程 (Manager)

### 2.1 企业准入审核

- [ ] 查看待审核企业列表
- [ ] 审核企业申请详情
- [ ] 通过准入（pending_review → approved）
- [ ] 驳回准入（pending_review → rejected）
- [ ] 企业状态管理（停用 suspended / 锁定 locked / 过期 expired）
- [ ] 6状态完整流转验证

### 2.2 审计批次管理

- [ ] 创建审计批次（名称、年份、描述、填报截止、审核截止）
- [ ] 设置业务类型（能源审计 / 节能诊断）
- [ ] 批次列表查看与筛选
- [ ] 批次详情查看

### 2.3 企业分配与项目创建

- [ ] 选择企业分配到批次
- [ ] 批量创建审计项目
- [ ] 项目自动绑定模板版本
- [ ] 企业信息快照生成

### 2.4 项目生命周期管理（12状态）

- [ ] pending_start → configuring（启动项目）
- [ ] configuring → filing（配置完成，进入填报）
- [ ] filing → pending_submit（填报完成）
- [ ] pending_submit → pending_report（提交确认）
- [ ] pending_report → report_processing（触发报告生成）
- [ ] report_processing → pending_review（报告就绪）
- [ ] pending_review → in_review（分派审核）
- [ ] in_review → pending_rectification（审核完成，需整改）
- [ ] pending_rectification → in_rectification（整改开始）
- [ ] in_rectification → completed（整改验收通过）
- [ ] completed → closed（项目关闭）
- [ ] 状态转换前置条件校验
- [ ] 状态转换历史记录

### 2.5 业务类型配置

- [ ] 能源审计（energy_audit）模块配置
- [ ] 节能诊断（energy_diagnosis）模块配置
- [ ] 模块可见性控制
- [ ] 模块必填性控制

### 2.6 审核任务管理

- [ ] 分派审核任务给审核员
- [ ] 查看审核进度
- [ ] 确认审核结论
- [ ] 退回审核（要求重审）

### 2.7 整改督办

- [ ] 查看整改任务列表
- [ ] 整改进度跟踪
- [ ] 超期预警
- [ ] 整改验收确认
- [ ] 整改闭环关闭

### 2.8 统计分析

- [ ] 批次统计（完成率、超期率、平均得分）
- [ ] 行业统计（行业能耗分布、达标率）
- [ ] 碳排放统计（总量、结构、趋势）
- [ ] 企业排名与历年趋势

### 2.9 台账管理与导出

- [ ] 企业台账（企业名称、项目状态、能耗指标汇总）
- [ ] 审核台账（任务完成情况、评分汇总）
- [ ] 整改台账（任务状态、完成情况汇总）
- [ ] CSV/Excel 导出功能
- [ ] 台账筛选与排序

### 2.10 首页管理看板

- [ ] KPI 卡片展示（批次进度、企业完成率、审核通过率、整改闭环率）
- [ ] 进度看板
- [ ] 异常预警列表（超期项目、未提交企业等）
- [ ] 最近操作时间线

## 五、跨端业务流程验证

### 5.1 完整审计流程（端到端）

- [ ] 管理端创建批次 → 分配企业 → 创建项目
- [ ] 企业端配置基础数据 → 填报24模块 → 提交
- [ ] 系统生成报告初稿
- [ ] 管理端分派审核任务
- [ ] 审核端评分 → 登记问题 → 提交结论
- [ ] 企业端接收整改 → 认领 → 更新进度 → 提交验收
- [ ] 管理端确认整改 → 项目关闭

### 5.2 业务类型差异验证

- [ ] 能源审计批次：全部24模块可见
- [ ] 节能诊断批次：模块按配置过滤
- [ ] 不同业务类型使用不同报告模板
